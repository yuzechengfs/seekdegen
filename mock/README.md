# Mock payloads

These files are example API payloads for the `seekdegen` prototype. They are useful for frontend development, sample contract review, and fallback local mocking.

Suggested mappings include:

- `me-kai.json` -> `GET /api/v1/me`
- `persona-questions.json` -> `GET /api/v1/persona/questions`
- `persona-answers-result.json` -> `POST /api/v1/persona/answers`
- `signals-today-pending.json` -> `GET /api/v1/signals/today`
- `signals-answer-success.json` -> `POST /api/v1/signals/answers`
- `home-kai.json` -> `GET /api/v1/home`
- `recommendations-kai.json` -> `GET /api/v1/recommendations`
- `candidate-linlan.json` -> `GET /api/v1/candidates/u_demo_linlan`
- `responses-incoming.json` -> `GET /api/v1/responses/incoming`
- `story-rooms.json` -> `GET /api/v1/story-rooms`
- `story-room-detail-active.json` -> `GET /api/v1/story-rooms/sr_demo_001`
- `story-answer-complete.json` -> `POST /api/v1/story-rooms/sr_demo_001/answers`
- `chats-list.json` -> `GET /api/v1/chats`
- `chat-thread-anyu.json` -> `GET /api/v1/chats/ct_demo_001`
- `chat-message-sent.json` -> `POST /api/v1/chats/ct_demo_001/messages`
